# Spooklight Vision

Spooklight is a collaborative worldbuilding system. Its purpose is to help a user create, grow, organize, and refine fictional worlds over time.

The core idea is that a world is not just a folder for notes. A world is the top-level container for everything the user creates: facts, aesthetics, characters, locations, important events, references, generated images, uploaded art, inferred relationships, inspirations, and story concepts. The system should help those pieces accumulate into a coherent body of truth that can be referenced later and used to generate new ideas that still feel native to the world.

Spooklight should feel like a world truth builder and manager. The user remains the author, while the AI acts as a collaborator that helps clean up rough inputs, connect ideas, suggest implications, and generate coherent new material from the world that already exists.

## Worlds Are the Entry Point

The top-level object in Spooklight is the world.

From the dashboard, the user's primary action is to create or open a world. A user can own multiple worlds, and each world is a complete private container for the material inside it.

Starting a new world begins the process of building from a seed. Over time, that world can grow into a rich collection of structured and semi-structured artifacts. Every feature should be scoped through the current world so that facts, references, images, aesthetics, and stories stay coherent and isolated from other worlds.

## World Truths

Within a world, the user should be able to add facts or ground truths as standalone units of information.

These truths are similar to pieces of a world bible. They might describe a place, faction, character, custom, law of nature, historical event, object, belief system, visual motif, or any other meaningful part of the world. They should be easy to add piecemeal, without forcing the user to decide up front where every idea belongs.

The system should help refine these truths after creation. It can clean up rough notes, clarify wording, associate related ideas, and make the truth referenceable later. The user should be able to write manually, accept AI-refined versions, reject changes, or keep multiple useful interpretations.

World truths should be stored in a way that supports semantic retrieval. A likely implementation path is Postgres with vector search, so truths can be embedded and retrieved dynamically when the user generates related material, develops stories, creates images, or asks the system to infer new ideas.

## Core World Elements

Some pieces of a world should become first-class structured elements rather than staying only as loose truths.

Characters, locations, and important events are especially important top-level concepts inside a world. They should still be scoped to a world, but each should have its own data model, fields, relationships, and associated images or references.

Characters should be able to capture details such as role, identity, appearance, personality, affiliations, motivations, secrets, relationships, history, and current narrative status. A character can have generated or uploaded images that represent them literally, explore visual variations, or act as inspiration.

Locations should be able to capture details such as location type, geography, visual character, culture, inhabitants, controlling factions, history, mood, hazards, and significance to the world. Locations should also support associated images, including maps, environmental art, architecture references, and generated concepts.

Important events should be able to capture details such as time period, participants, causes, consequences, affected locations, related factions or characters, and whether the event is historical, current, predicted, mythical, or disputed. Events can also have associated images or references, such as symbolic art, documentary-style scenes, artifacts, maps, or timelines.

These elements should not replace freeform world truths. Instead, they should provide structure for the concepts that users naturally return to and reuse. A truth might eventually become associated with a character, location, event, image, story concept, or several of these at once.

## Images and References

Each world should have an images and references area for visual material.

Users should be able to create new images with AI image generation, guided by the world's aesthetic principles and relevant world truths. Generated art should not be generic. It should draw from the world's established style, facts, locations, characters, story elements, and references where appropriate.

Users should also be able to upload or drop in their own art and reference material. User-provided images should be tracked separately from generated images. Their metadata should make clear whether an image was generated by the system or provided by the user.

Images should become first-class artifacts inside the world. They should have editable metadata, descriptive context, and a role in the world. For example, an image might be:

- An inspirational reference that influences the style or mood of later work.
- A literal artifact that depicts something that exists in the world.
- A generated concept that the user accepts, edits, or rejects.

Image metadata and descriptions should also be embedded for semantic retrieval. This allows visual references to be pulled into context later when generating related art, expanding facts, or developing stories.

## Aesthetics, Genre, and Archetypes

Each world needs an overarching layer that defines its dominant aesthetic, genre, archetypes, and core storytelling truths.

This layer is separate from individual facts. It describes the broad nature of the world: its visual language, tonal principles, genre expectations, recurring motifs, and the assumptions that should influence everything else created inside it.

For example, a Lovecraftian world might be grounded in cosmic horror. A riverpunk world might define an aesthetic of sepia-toned late-1800s photography, rundown villages on stilts, steamships, steam-operated machinery, river trade, and stone temples to mysterious gods.

These aesthetic principles should flow downward into the rest of the system. When the user generates art, asks for new world truths, or develops story concepts, the system should be able to use the world's broad aesthetic seed without requiring every prompt to restate it.

Specific facts and references can still be incorporated when relevant, but the world should always have a higher-level creative identity that guides everything built inside it.

Whenever AI is used to generate or refine anything inside a world, the generation process should consider the world's aesthetics as required context. Depending on what is being generated, it should also retrieve and incorporate relevant characters, locations, important events, truths, images, references, and story concepts. The goal is not to stuff every output with every fact, but to ensure generated material is grounded in the parts of the world that matter for that request.

## Inference and Inspiration

Spooklight should help users expand a world by inferring relationships and implications between existing pieces of information.

The user should be able to select one or more truths, images, story elements, or references and ask questions such as:

- What does this imply?
- What related ideas could grow from this?
- What tensions, conflicts, or consequences emerge here?
- What new facts would make this part of the world richer?

When the AI generates ideas, it should usually produce multiple variations. At least three options is a good default. The user can accept none, one, several, or all of them.

Accepted ideas become part of the world. Rejected ideas disappear. This preserves the user's authorship while still letting the AI push the world's inspiration outward.

## Story Concepts

Stories should be distinct artifacts inside a world.

Spooklight is not primarily about generating complete stories in one shot. Instead, it should help create story concepts that live within an established world.

The user should be able to write a seed idea and ask the AI to expand it using the truths, aesthetics, references, and existing story material in the world. The user should also be able to ask the AI to look across the world's accumulated material and propose story concepts that could naturally emerge from it.

Story concepts should stay connected to the world that produced them. They should be able to reference relevant truths, images, characters, locations, and aesthetic principles, while still remaining separate artifacts that can be developed independently.

## The AI's Role

The AI should act as a collaborative worldbuilding partner, not as the owner of the creative work.

Its responsibilities include:

- Cleaning up rough notes and turning them into usable world truths.
- Associating related facts, characters, locations, events, references, images, and story concepts.
- Retrieving relevant context from the world before generating new material.
- Suggesting implications, variations, and expansions.
- Helping new material remain coherent with the world's established aesthetic and truths.
- Creating optional alternatives so the user can choose what belongs.

The user should always be able to write manually, override AI suggestions, reject generated ideas, and decide what becomes canonical inside the world.

## Long-Term Shape

A world should be able to grow and evolve indefinitely.

The more the user adds, the more useful the system becomes. Spooklight should aggregate the world's truths, images, references, aesthetics, and story concepts into a living creative substrate. Over time, that substrate should make the world more coherent, more searchable, and more generative.

The product should support both deliberate organization and loose inspiration. A user can carefully define facts, upload reference art, and tune aesthetic principles. They can also drop in fragments, ask the AI what those fragments imply, and accept the ideas that feel right.

The outcome is a private creative workspace where worlds become richer through an ongoing collaboration between the user's imagination and the system's ability to organize, retrieve, infer, and generate.
